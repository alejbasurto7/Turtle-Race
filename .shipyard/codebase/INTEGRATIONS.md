# INTEGRATIONS.md

## Overview

Turtle Race has no network integrations. All external dependencies are local file assets (images, MIDI) and OS-level subsystems (the system MIDI synthesizer accessed via `pygame.mixer`). The PyInstaller packaging layer adds a second resolution path (`sys._MEIPASS`) that all asset access must route through. There are no databases, external APIs, or message queues.

## Metrics

| Metric | Value |
|--------|-------|

## Findings

### Asset Resolution

- **`resource_path()` helper** — all asset file paths in the application must be resolved through this single function. It detects PyInstaller's frozen-bundle mode via `sys._MEIPASS` and falls back to the directory of `paths.py` during normal development runs.
  - Evidence: `paths.py` lines 1–7
  - Pattern: `resource_path("assets/TeenageMutantNinjaTurtles.mid")` — relative paths only, never absolute.
- Every consumer calls `resource_path` before opening any file: `audio.py` line 9, `dialogs.py` lines 42, 101, `race.py` line 52.
- **PyInstaller data bundling** — the spec declares four asset globs. If a new asset file is added without updating the spec, it will load from source but silently fail in the packaged exe.
  - Evidence: `turtle_race.spec` line 7: `datas=[('lawn.jpg', '.'), ('assets/*.jpg', 'assets'), ('assets/*.mid', 'assets'), ('assets/*.png', 'assets')]`

### Image Assets

- **Background image (`lawn.jpg`)** — a lawn photograph in the project root. Loaded at the start of every round in `race.set_background()`. Scaled and center-cropped to fill the full window via Pillow, then pushed to the Tk canvas as the bottom-most item. The `ImageTk.PhotoImage` reference is stashed on `canvas._bg_photo` to prevent Tk garbage collection.
  - Evidence: `race.py` lines 48–62
- **Turtle character images** — 4 JPEG files in `assets/`, one per turtle. File names encode the bet-dialog grid position (`top_left`, `top_right`, `bottom_left`, `bottom_right`).
  - Evidence: `constants.py` lines 19–24

  | Turtle | File |
  |--------|------|
  | Raphael | `assets/turtle_3_raphael_bottom_left.jpg` |
  | Michaelangelo | `assets/turtle_4_michelangelo_bottom_right.jpg` |
  | Leonardo | `assets/turtle_1_leonardo_top_left.jpg` |
  | Donatello | `assets/turtle_2_donatello_top_right.jpg` |

- **Track preview images** — 3 PNG files in `assets/`, shown in the track-selection dialog. These are generated by `tools/generate_track_previews.py` and committed to the repo; they are not generated at runtime.
  - Evidence: `dialogs.py` lines 94–98; `tools/generate_track_previews.py` lines 181–192

  | Track | File |
  |-------|------|
  | Straight | `assets/track_straight.png` |
  | Rectangular | `assets/track_rectangular.png` |
  | Spiral | `assets/track_spiral.png` |

- **`ImageTk.PhotoImage` lifetime management** — Pillow images converted for Tk display must be kept alive for as long as the widget exists. The bet dialog stashes them on `dialog._bet_images` (a list attribute); the track dialog stashes them on `dialog._track_images`.
  - Evidence: `dialogs.py` lines 37, 45, 92, 102

### Audio

- **MIDI file** — `assets/TeenageMutantNinjaTurtles.mid` is the sole audio asset. Played as looped background music via `pygame.mixer.music`.
  - Evidence: `main.py` line 16; `audio.py` lines 9–11
- **`pygame.mixer`** — initialized once at application startup (`pygame.mixer.init()`). `pygame.mixer.music.play(loops=-1)` streams the MIDI file indefinitely. Stopped via `pygame.mixer.music.stop()` on exit.
  - Evidence: `audio.py` lines 8–11, 16
- **OS MIDI synthesizer dependency** — `pygame.mixer` on Windows delegates MIDI playback to the system's GS Wavetable Synthesizer (or whatever MIDI device is available). If no MIDI output device is present, music silently fails; the `except Exception` block in `audio.start_background_music()` swallows the error.
  - Evidence: `audio.py` lines 11–12; `tools/diagnose_audio.py` lines 52–68 (tests `winmm.midiOutOpen` directly)
- [Inferred] MIDI playback reliability on non-Windows platforms is unknown. The diagnostic tool (`tools/diagnose_audio.py`) only tests Windows-specific audio paths.

### Tk Canvas Integration (turtle + tkinter coexistence)

- The `turtle.Screen()` canvas is a Tk widget. Both `race.py` and `dialogs.py` call `_screen.getcanvas()` to draw directly via the underlying `tkinter.Canvas` API (for background image placement, checkered finish line, and podium medals), bypassing the turtle drawing model.
  - Evidence: `race.py` lines 52–61 (background), 111–124 (finish checker), 313–345 (podium medals)
- The `turtle` module raises every turtle sprite to the top of the canvas stacking order on each `_screen.update()` call. This conflicts with manually-drawn canvas items (medals, background). The code works around it by explicitly calling `canvas.tag_raise(PODIUM_MEDAL_TAG)` after each update.
  - Evidence: `race.py` lines 345, 365, 422

### Dev Tooling Integrations

- **`tools/generate_track_previews.py`** — a one-shot script that writes PNG files to `assets/`. Imports Pillow and the project's `constants.py`. Not invoked during normal app execution or tests.
  - Evidence: `tools/generate_track_previews.py` lines 1–7, 181–192
- **`tools/diagnose_audio.py`** — a diagnostic script that tests Windows audio paths (`winsound`, `winmm` via `ctypes.windll`, `pygame.mixer`). Uses Tk dialogs for results. Not part of the test suite.
  - Evidence: `tools/diagnose_audio.py` lines 1–9

## Open Questions

- Is there an intention to support sound effects beyond the looped MIDI track? The `audio.py` module only wraps `pygame.mixer.music`; `pygame.mixer.Sound` (for one-shot sounds) is not used.
- The track preview PNGs are generated by a dev tool and committed to the repo. If the window size constants in `constants.py` change, the previews become stale. There is no CI step to regenerate them.
- `pygame.mixer.init()` is called with no arguments, so it uses pygame's defaults for frequency, bit depth, and buffer size. If the system MIDI device is slow or absent, there may be a perceptible delay on startup or silent failure — the error is caught but not surfaced to the user.
